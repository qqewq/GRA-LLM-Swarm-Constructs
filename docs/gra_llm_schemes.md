# Схемы: конструкты LLM ↔ агенты GRA‑роя

## 1. Архитектурная схема

```
+-------------------+     +-----------------+
|   Input tokens    |     | External RAG/   |
+--------+----------+     | Verifier        |
         |                +--------+--------+
         v                         |
+--------+----------+              |
| Embedding + Pos   |              |
+--------+----------+              |
         |                         |
         v                         |
+--------+------------------+      |
| Transformer Layer 0       |<-----+
| ┌─────────────────────┐   | (hallu foam)
| │   Attn Constructs   │   |
| └─────────────────────┘   |
| ┌─────────────────────┐   |
| │   Plan Constructs   │   |
| └─────────────────────┘   |
+--------+------------------+
         | Φ₀, ΔΦ₀
         v
+--------+------------------+
| Transformer Layer 1       |
|           ...             |
+--------+------------------+
         |
         v
         ...
         |
         v
+--------+------------------+
| Transformer Layer L       |
| ┌─────────────────────┐   |
| │  High‑Level Planner │   |
| └─────────────────────┘   |
+--------+------------------+
         | Φ_L, Ψ
         v
+--------+--------+
|   Output Head   |
+-----------------+
```

Каждый слой содержит «рой конструктов» (attn‑головы, подпространства FFN, планировщики).  
Пена вычисляется послойно, штрафы \( C_l \) — между соседними слоями.

## 2. Роевая структура внутри одного слоя

```
Layer l:
+--------------------------------------------------+
| Construct A (Attn head)                          |
|   state z_A                                      |
|   foam Φ_A                                       |
|   love L_AB                                      |
+------------------------+                         |
| Construct B (Plan)     |<-- foam_pair A-B       |
|   state z_B            |    nullify if > thr     |
+------------------------+                         |
| Construct C (Memory)   |                         |
+------------------------+-------------------------+
```

Каждый конструкт может быть отдельным агентом в алгоритме GRA‑Swarm‑Optimization:
- A критикует B, находит фундаментальные ошибки;
- Применяются операторы \( T_{A\to B} \) — локальная коррекция весов/активаций.

## 3. Поток GRA‑шага

```
┌─────────────┐
│ Forward pass│
└─────┬───────┘
      │ h^(0)..h^(L)
      v
┌─────────────┐
│ Compute Φ_l │
│ Compute C_l │
└─────┬───────┘
      │ Φ, ΔΦ, Δ²Φ
      v
┌────────────────────┐ No  ┌──────────────┐
│ Stable?            ├───> │ Output logits│
│ Φ=0, ΔΦ=0, Δ²Φ>0? │     └──────────────┘
└─────┬──────────────┘
      │ Yes
      v
┌─────────────────────┐
│ Extract constructs  │
│ Compute foam_pairs  │
│ Apply Love‑nullify  │
│ Apply Tₗ operators  │
└─────────┬───────────┘
          │
          └──> return to forward pass (refined h)
```

## 4. Интеграция с GRA‑Multiverse

Один и тот же промпт обрабатывается в нескольких «мирах» LLM (разные LoRA, промпт‑префиксы).  
Мультиверсная обнулёнка (GRA‑Multiverse‑Final) выбирает мир с минимальной глобальной пеной.  
В каждом мире — локальные GRA‑шаги.

## 5. Используемые GRA‑репо

- `GRA-Swarm-Optimization` → роевая критика конструктов.
- `GRA-Hormonal-Explosion-Of-Subject` → гормональные уровни (веса важности конструктов).
- `GRA-Living-Vaccine-Architecture` → механизм иммунитета (удаление вредных конструктов).
- `GRA-Core-new-Unified-Hierarchical-Stability-Library` → core‑система с J и GRA‑шагом.
- `GRA-Love-Oriented-Nullification` → мягкое обнуление.
- `GRA-ASI-Metric-Space` → метрика расстояний конструктов.
- `GRA-Multiverse-Final` → мультиверсная оболочка.
